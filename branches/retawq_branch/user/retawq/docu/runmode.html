Runmodes

retawq can be run in any of the following modes. The desired runmode is detected when the program is started, doesn't change afterwards as long as the program runs and depends on command-line options. The most usual mode is probably the "curses-like" mode.

  • console: a "raw" text console which doesn't use any curses library support; similar to those very old shell-like FTP clients, but more flexible; cf. Custom Connections; this runmode is activated with the command-line option "--console"; interactive
  • curses-like: this is the default mode when none of the others is explicitly activated; it provides a "usual" text-mode web browser: the functionality of a curses/ncurses library or emulation is used (as specified with the compile-time configuration option OPTION_TG), and you can work interactively
  • download: this runmode is activated with the command-line option "--download"; automated, non-interactive
  • dump: this runmode is activated with the command-line option "--dump"; automated, non-interactive

Planned, not yet there:

  • GTK: using the GTK library; interactive
  • X: using pure X Window System; interactive
